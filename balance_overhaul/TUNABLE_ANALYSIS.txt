TUNABLE ANALYSIS — GROUNDED REVIEW OF ALL MG TUNABLES
================================================================================

METHOD
------
Each tunable was checked against the REAL game constants and formulas:
  - Income  : minutes * (mine_lvl * 5) gold/min, passive, offline-capped 24h
              (database.py:188-195). Mine L1 = 7,200 gold/day.
  - Cost    : base * 1.6^level  (game_logic.py:34)  -- exponential, NO level cap.
  - Combat  : simulate_battle (game_logic.py:51-180). Defender wins mutual
              annihilation and value-ties -> structural defender bias.
  - Constants: config.py (unit costs, upkeep, loot 30%, vault 1000/lvl, etc.)
NOTE: this workspace cannot execute Python (only ls/dbt/snow), so results were
derived analytically, not by running a sim.

CONFIDENCE-INTERVAL BASIS (used for all SIM_COUNT decisions)
  For a win-rate p~=0.5, SE = sqrt(0.25/n), 95% CI = 1.96*SE:
    n=200  -> +/-6.9%   (too wide to resolve a 50% vs 55% band)
    n=1000 -> +/-3.1%
    n=2500 -> +/-2.0%

KEY ECONOMY INSIGHT
-------------------
Income is LINEAR in mine level (5/min per level) but upgrade cost is EXPONENTIAL
(1.6^level). Late game therefore plateaus hard: each extra level of income costs
~1.6x more than the last. This single relationship drives MG2 (honest long
horizon) and MG3 (bounded summit) and must be the first thing tuned in the
overhaul.
