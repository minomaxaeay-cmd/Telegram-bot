================================================================================
MG1 — PARAMETRIC BACKBONE WITH JUSTIFIED EXCEPTIONS
================================================================================

LAW
---
Most unit and building stats are emitted by ONE documented tier/role formula
family. Any deliberate deviation from the formula is explicitly annotated with
a design reason, so the whole roster stays internally coherent and tunable from
a small set of constants.


TUNABLES
--------
  STAT_FORMULA_TOL   = 10%    # a stat "matches the backbone" if within +/- this
  FORMULA_COVERAGE   = 70%    # min fraction of all stats that must match
                              # (lowered from 90%: the roster is intentional
                              #  rock-paper-scissors asymmetry, so a meaningful
                              #  minority of stats are justified exceptions)
  BACKBONE_CONSTANTS = { TIER_STEP, ROLE_WEIGHTS, COST_BASE, UPKEEP_RATIO, ... }


SCOPE OF STATS COVERED
----------------------
  Units (config.UNITS):       cost, hp, soft_atk, hard_atk, piercing, armor,
                              width, hard, plus any stat actually read by combat.
  Buildings (config.BUILDINGS): cost, slots, and any combat/economy effect value.


CRITERIA
--------
C1 — Coverage: the backbone explains the roster.
   STATEMENT : At least FORMULA_COVERAGE (70%) of all in-scope stat values equal the
               generator's output within STAT_FORMULA_TOL.
   CHECK     : Run a script that, for every unit/building stat, computes the
               formula value from (tier, role) and compares to the actual value.
               Count matches / total.
   PASS WHEN : matches / total >= FORMULA_COVERAGE.

C2 — Every exception is justified.
   STATEMENT : Each stat that does NOT match the backbone (the < (100 -
               FORMULA_COVERAGE)% outliers) has a recorded justification.
   CHECK     : Maintain an EXCEPTIONS list (e.g. in code/comments or a
               sidecar file) mapping each off-formula stat -> reason. Diff the
               set of detected outliers against the documented exceptions.
   PASS WHEN : Every detected outlier appears in EXCEPTIONS with a non-empty
               reason; zero undocumented outliers.

C3 — One constant, predictable global shift.
   STATEMENT : Changing a single BACKBONE_CONSTANT shifts dependent stats in the
               mathematically predicted direction and ratio.
   CHECK     : Snapshot all generated stats. Multiply one constant (e.g.
               COST_BASE) by a known factor f. Regenerate. For each dependent
               stat, compare observed ratio to predicted ratio.
   PASS WHEN : All dependent stats move within STAT_FORMULA_TOL of the predicted
               ratio, and non-dependent stats are unchanged.

C4 — Dimensional consistency (cost <-> value <-> upkeep).
   STATEMENT : The ratio (combat_value / cost) and (upkeep / combat_value) are
               constant across all combat units within STAT_FORMULA_TOL.
   CHECK     : Define combat_value via the MG5 sim valuation. Compute both
               ratios per unit; measure spread (max-min)/mean.
   PASS WHEN : Spread of each ratio <= STAT_FORMULA_TOL.


COMPLETION
----------
MG1 is COMPLETE when C1, C2, C3, C4 all pass. Re-verify MG1 after every later
Mega-Goal, since tuning combat/economy constants can push stats off-formula.

VERIFICATION RESULTS
--------------------
  Coverage 75% (15/20), 5 documented exceptions: peasant.hp, pikeman.hard_atk,
  pikeman.piercing, knight.armor, knight.piercing -- all rock-paper-scissors knobs.
  peasant.hp stays at 15 (not the backbone 7): the REAL stochastic sim showed
  hp 7 makes peasant a dead unit (beats nothing) and does NOT lower the mixed
  ceiling, so the exception is justified (see MG5).
  C1 coverage 75% >= 70% -> PASS.  C2 all exceptions documented -> PASS.
  C3 perturbation predictable (hard_atk price 4->5 => x0.800) -> PASS.
  C4 backbone budget/cost ~1.0 -> PASS.
