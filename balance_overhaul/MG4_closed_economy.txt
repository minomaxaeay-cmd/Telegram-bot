================================================================================
MG4 — CLOSED CONSERVATIVE ECONOMY (NO INFLATION)
================================================================================

LAW
---
The economy is a closed, conservative system. Gold and troops are never created
or destroyed except through DECLARED sources and sinks:
  SOURCES : gold mines (income).
  SINKS   : army upkeep, troops lost in battle, desertion, building costs,
            recruitment costs, and the alliance-founding fee.
Player-vs-player transfers (loot, heist) are pure transfers — zero net creation.


TUNABLES
--------
  GOLD_RECONCILE_TOL = 1 gold     # gold is a FLOAT (database.py: gold REAL).
                                  # Exact-0 reconciliation fails on rounding, so
                                  # allow a 1-gold (or 1e-6 relative) tolerance.
  SIM_PLAYER_YEARS   = 50         # total simulated player-time for the long run
  DECLARED_SOURCES   = { gold_mine_income }
  DECLARED_SINKS     = { upkeep,                # database.py: net = income - upkeep
                         battle_casualties,     # troops removed in simulate_battle
                         desertion,             # database.py: unpaid troops desert
                         building_cost,         # game_logic.can_build cost is spent
                         recruitment_cost,      # gold spent buying units
                         alliance_fee }         # database.py:428 (2000 gold) vanishes


CRITERIA
--------
C1 — Transfers are zero-sum.
   STATEMENT : Every battle loot and every spy heist removes exactly as much gold
               from the loser as it grants the winner (after the vault floor and
               carry caps are applied) — no gold appears or vanishes.
   CHECK     : Instrument loot/heist transfers in a sim; for each event record
               (delta_winner + delta_loser).
   PASS WHEN : For every transfer event, delta_winner + delta_loser == 0.

C2 — Global ledger reconciles.
   STATEMENT : Over a long multi-player simulation, total gold in the world
               equals (sum of all mine output) minus (sum of all sinks).
   CHECK     : Run a multi-agent sim for SIM_PLAYER_YEARS. Track a running ledger:
               start_gold + Σ(mine income) - Σ(sinks) - Σ(net transfers=0).
   PASS WHEN : |observed_total_gold - ledger_predicted_total| <= GOLD_RECONCILE_TOL.

C3 — No runaway inflation.
   STATEMENT : Long-run growth of total world gold is bounded by mine output;
               there is no exponential / unbounded accumulation channel.
   CHECK     : Plot total world gold over time in the long sim; fit growth rate.
   PASS WHEN : Total-gold growth rate <= total mine output rate at all times
               (no super-linear blowup).

C4 — Troops are conserved (no phantom units).
   STATEMENT : Troop counts only decrease via casualties/desertion and only
               increase via paid recruitment; battles never create units.
   CHECK     : In sim, assert post-battle troop counts = pre-battle - losses for
               both sides; recruitment always debits the correct gold cost.
   PASS WHEN : No troop count ever increases as a result of a battle or heist,
               and every recruit event has a matching gold debit.


COMPLETION
----------
MG4 is COMPLETE when C1, C2, C3, C4 all pass. After completing MG4, re-verify
MG1, MG2, MG3.

MULTI-CURRENCY EXTENSION (Iron / Grain / Faith)
-----------------------------------------------
The economy now has FOUR currencies. The closed-conservative law applies to each
independently. Buildings remain GOLD-ONLY priced, so the gold horizon math
(MG0/MG2/MG3) is unchanged; the new currencies attach to the army economy.

  CURRENCY  DECLARED SOURCE          DECLARED SINKS                         TRANSFERS
  gold      gold-mine income         building, recruitment, gold upkeep,    loot, heist
                                      treasury floor, alliance fee           (zero-sum)
  iron      iron-mine income         military recruitment (IRON_COST_PER_   none
            (iron_mine*2/min)         UNIT); future: siege engines
  grain     farm income              food upkeep (GRAIN_UPKEEP_PER_UNIT);    none
            (farm*6/min)             grain floored on starvation
  faith     chapel income            blessings (BLESSING_DEFS faith cost)    none
            (chapel*0.5/min)

  Desertion (gold OR grain shortfall) destroys UNITS, not currency, and is a single
  bounded event per tick (<= DESERTION_CAP, scaled by the worse shortfall).
  Iron/Grain/Faith have NO PvP transfer path, so they are trivially zero-sum.

  VERIFICATION: criteria_evaluations/simulate_currency_ledger.py runs the REAL
  database income tick + real sinks over a multi-lord, multi-tick scenario and
  reconciles each currency: observed_total == start + Σsources - Σsinks within
  GOLD_RECONCILE_TOL (1.0). RESULT: gold/iron/grain/faith all reconcile (residuals
  < 0.15, float noise); transfer_net == 0. MG4 holds for all four currencies.
