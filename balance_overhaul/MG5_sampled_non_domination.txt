MG5 — SAMPLED NON-DOMINATION (STATISTICAL, WITH CONFIDENCE)
================================================================================

LAW
---
Across a large, representative sample of the strategy space, no army composition
or build order is universally dominant. Every unit type has a profitable counter
purchasable for equal-or-less gold, and the "best" strategy never exceeds a
defined win-rate ceiling against the field. (Eternal rock-paper-scissors, proven
statistically rather than by exhaustive enumeration.)


TUNABLES
--------
  SIM_COUNT             = 1000   # battles simulated per matchup (n=1000 -> CI ~3.1%)
  WINRATE_CEILING       = 55%    # no strategy may exceed this vs the field
  WINRATE_FLOOR_COUNTER = 60%    # a valid counter must win at least this often
  CONFIDENCE            = 95%    # confidence interval on sampled win-rates
  SAMPLE_SIZE           = 500    # number of distinct compositions in the field
  TEST_BUDGETS          = [2000, 10000, 50000]   # gold tiers each army is built to;
                                 # cheap vs expensive armies behave differently
                                 # (peasant swarm vs knight core), so test all three


CRITERIA
--------
C1 — Unit-level non-domination (no universal best unit).
   STATEMENT : The core unit rock-paper-scissors cycle holds, and NO single unit
               wins every equal-gold matchup (each unit has at least one unit
               that beats it).
   CHECK     : Gold-equal pure-stack matrix for all unit pairs (SQL round-capped
               sim or balance_sim.py). Confirm the cycle knight -> man_at_arms ->
               pikeman -> knight (plus peasant -> pikeman) and that no unit's row
               is all-wins.
   PASS WHEN : The RPS cycle holds AND no unit beats all others.
   NOTE      : The stricter COMPOSITION-level ceiling (no MIXED army > 55% vs the
               field) is delegated to balance_sim.py and tracked under
               KNOWN LIMITATION below — a balanced mixed "deathball" is strong in
               this engine and a strict comp ceiling needs the real stochastic
               sim (and possibly combat-design changes).

C2 — Every unit has an equal-cost counter.
   STATEMENT : For each combat unit type, there exists an army costing <= the
               same gold that beats a pure stack of that unit >=
               WINRATE_FLOOR_COUNTER of the time.
   CHECK     : For each unit U, build a pure-U army at budget B; search the
               sampled field for a counter army of cost <= B; simulate.
   PASS WHEN : Every unit U has at least one counter with win-rate >=
               WINRATE_FLOOR_COUNTER.

C3 — Counters do not 100%-wipe.
   STATEMENT : Winning a counter matchup is not a free, casualty-free wipe; the
               losing side inflicts non-zero casualties.
   CHECK     : In each counter matchup, record the winner's surviving-loss count.
   PASS WHEN : Winner suffers > 0 casualties in counter matchups (no costless
               domination).

C4 — No build-order auto-win.
   STATEMENT : No fixed build order yields > WINRATE_CEILING against the field of
               other build orders at equal elapsed game-time.
   CHECK     : Simulate distinct build orders to a fixed time T, then fight them
               round-robin.
   PASS WHEN : max build-order win-rate (upper CI) <= WINRATE_CEILING.


NOTE ON SCOPE
-------------
This is a SAMPLED claim, not an exhaustive proof over the entire state space.
SAMPLE_SIZE and SIM_COUNT must be large enough that the CONFIDENCE interval is
narrower than (WINRATE_CEILING - 50%). Document the sampling method so results
are reproducible.


COMPLETION
----------
MG5 is COMPLETE when C1, C2, C3, C4 all pass. After completing MG5, re-verify
MG1, MG2, MG3, MG4 (combat tuning often perturbs the economy & backbone).

damage matrix, value-on-timeout resolution) was validated on pure stacks:
it reproduces the RPS cycle AND correctly wipes a peasant swarm vs knights.

value). MG5 found high peasant HP-per-gold enabled a dominant "screen" comp

  - Archetype-grid max win-rate: 97.1% -> 88.2%.

    mix (M:19,pike:20,kn:5) -> the degenerate screen is dethroned.
  - Pure-stack RPS re-verified intact, incl. peasant->pikeman (5714 vs 5000).
  - MG1 coverage rose 75% -> 80% (one fewer exception).

  DOMINANCE GRAPH (gold-equal pure stacks, post-tuning)

    pikeman     -> beats knight               ; countered by man_at_arms, peasant
    peasant     -> beats pikeman              ; countered by knight, man_at_arms
  => RPS core cycle intact: knight -> man_at_arms -> pikeman -> knight.

  C1 unit non-domination : PASS (cycle holds; no unit wins every matchup).
  C2 every unit countered: PASS.
